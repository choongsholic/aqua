# DP-AQUA · Implementation Guide

## 개요
**DP-AQUA** (App User Quantitative Analysis)는 현대카드 앱 MAU 800만 달성을 위한 유저 행동 분석 시각화 대시보드.  
단일 HTML 파일(`dp-aqua.html`)로 구현된 Canvas 2D 기반 실시간 파티클 시뮬레이션.

## 컨셉
- 유저를 수조(Aquarium) 속 점으로 표현
- 세그먼트 간 이동, 이탈, 유입을 자연생태계처럼 시뮬레이션
- 마우스 호버 시 물고기 떼가 흩어지는 인터랙션
- **웹캠 손 추적**으로 파티클 제어 (MediaPipe Hands + Face Mesh)

## 기술 스택
- **순수 HTML + Canvas 2D** (프레임워크 없음)
- **MediaPipe Hands** — 손가락 추적, 제스처 인식
- **MediaPipe Face Mesh** — 얼굴 인식(최대 3명), AR 이펙트
- 폰트: YouandiNewKr (타이틀), SF Pro Display (본문)
- 단일 파일, 빌드 없음 — 브라우저에서 바로 실행

## 핵심 시스템

### 1. 파티클 (7,630개)
- **1만 단위**: 763만 MAU → 7,630개 점
- **분포**: Power 1,910 / Casual 2,670 / Light 3,050
- **크기**: 1.8px 통일
- **서브타입**: 각 세그먼트 내 TYPE 1(50%), TYPE 2(30%), TYPE 3(20%)

### 2. 건강 상태 (색상 체계)
| 상태 | 색상 | RGB | 분포 기준 |
|------|------|-----|----------|
| 건강 | 녹색 | 80,210,120 | Power 55%, Casual 8%, Light 2% |
| 양호 | 파란색 | 80,150,230 | Power 25%, Casual 28%, Light 6% |
| 일반 | 흰색 | 200,200,200 | Power 12%, Casual 35%, Light 22% |
| 주의 | 노란색 | 220,190,80 | Power 6%, Casual 22%, Light 40% |
| 위험 | 빨간색 | 220,75,75 | Power 2%, Casual 7%, Light 30% |

- 신규 유입은 항상 **일반(흰색)**으로 진입

### 3. 물리 엔진
```javascript
CFG.physics = {
  noise: 0.034,        // 유기적 노이즈 강도
  maxSpeed: 0.8,       // 기본 최대 속도
  maxSpeedScatter: 2.5, // 호버 시 흩어짐 최대 속도
  damping: 0.995,      // 감쇠 (1에 가까울수록 무중력)
  boundary: 0.01,      // 벽 반발력
  repulsion: 0.03,     // 파티클 간 척력
  mouseR: 150,         // 마우스 반발 반경
  mouseF: 2.0,         // 마우스 반발력
}
```

### 4. Boids 알고리즘
- **Cohesion**: 타원형 중심 방향으로 끌어당김 (거리 비례)
- **Alignment**: 주변 파티클과 같은 방향으로 이동 (20px 반경)
- **Swirl**: 접선 방향 회전력 → 소용돌이 군집 운동
- **Separation**: 파티클 간 척력 (공간 그리드 최적화)
- **서브존 벽 무시 금지**: 같은 세그먼트 다른 서브타입 간 척력 차단

### 5. 서브존 경계 시스템 (Hard Wall)
- TYPE 구분선 = **절대 벽** (어떤 상황에서도 통과 불가)
- 구분선은 `t.y+28` ~ `t.y+t.h-28` 구간에만 표시
- 상단/하단 채널(28px 미만 영역) = 마이그레이션 전용 통로
- **3중 방어**:
  1. 힘 적용 후 사전 clamp (위치 강제 + 속도 차단)
  2. update() 내 absolute clamp + 속도 반사
  3. 마이그레이션 완료 시 서브존 내 강제 배치

### 6. 마이그레이션 시스템
```javascript
CFG.migration = {
  speed: 0.004,
  rates: {
    light_casual: 0.03,   // Light→Casual 상향
    casual_power: 0.02,   // Casual→Power 상향
    power_casual: 0.012,  // Power→Casual 다운
    casual_light: 0.025,  // Casual→Light 다운
    churn: 0.08,          // 이탈 (Light 100%, Casual 40%, Power 15%)
  }
}
```

**규칙**:
- 상향(Up): 건강/양호 파티클만 선택
- 다운(Down): 주의/위험 파티클만 선택
- 이탈: 주의/위험 우선, 모든 세그먼트에서 발생

**경로 원칙** (서브존 벽 관통 금지):
- 모든 마이그레이션은 **상/하단 채널**(구분선이 없는 영역)을 경유
- 출발: 현재 위치 → 수직으로 채널 이동 → 수평으로 파이프까지 → 파이프 통과
- 도착: 파이프 출구 → 채널에서 수평으로 서브존 위치까지 → 수직으로 존 안으로
- Power/Casual 이탈: 세로 파이프 타고 Light까지 → Light 하단 채널 → 이탈 파이프

### 7. 파이프 시스템
- **세로 파이프**: Power↔Casual, Casual↔Light (Up/Down 쌍)
- **가로 파이프 (신규유입)**: 신규유입 영역 → Light 탱크 좌측벽 직접 연결
- **가로 파이프 (이탈)**: Light 탱크 우측벽 → 이탈 영역 직접 연결
- 파이프 폭: 38px
- Light 탱크 좌/우측벽에 파이프 전용 구멍
- Power/Casual은 우측 구멍만 유지 (즉시 이탈용 예비)
- 좌측 구멍/쉐브론: 모든 탱크에서 제거
- **그리기 순서**: 탱크 먼저 → 파이프 나중에

### 8. 신규유입 시스템
- 이탈 파티클 재활용 → 신규유입 영역에 **페이드인** 등장
- 40~100프레임 대기 후 마이그레이션 시작
- 경로: 신규유입 → 가로 파이프 → Light 좌측벽 구멍 → 하단 채널 → 서브존

### 9. 이탈 생태계
- 이탈 도착 시: 60% 페이드아웃(사라짐), 40% 체류 후 복귀
- 페이드아웃 속도: fadeAlpha -= 0.008/frame
- 복귀 타이머: 80~200 프레임
- 복귀 경로: 이탈 파이프 역방향 → Light 우측벽 구멍 → 하단 채널 → 서브존
- 사라진 파티클은 신규유입으로 재활용 (총량 유지)

### 10. 신규유입/이탈 영역
- 위치: 아쿠아리움 하단 정렬, 높이 160px 고정
- 신규유입: 좌측, 페이드인 효과
- 이탈: 우측, 빨간 테두리
- 파이프가 Light 탱크 벽에 직접 연결

### 11. 인터랙션 (마우스)
- **마우스 호버**: 파티클 흩어짐 (반경 150px, 랜덤 방향 scatter)
- **scattered 상태**: 30프레임 동안 maxSpeed 2.5배, 이후 자연 감속
- **드래그**: 아쿠아리움 내부 드래그 시 관성으로 파티클 흔들림
- **TYPE 텍스트**: 해당 영역 호버 시 투명도 48%→100%
- **캐러셀**: ACTION INSIGHT 패널 클릭으로 슬라이드 전환

### 12. 웹캠 손 추적 (MediaPipe Hands)
- 우하단 카메라 아이콘(32px, 마진 12px) 클릭으로 활성화
- 카메라 뷰: 드래그로 위치 이동, 우하단 핸들로 크기 조절 (min 160, max 480)
- X 버튼으로 카메라 끄기 (추적도 완전 중지, MediaPipe 처리 정지)
- 브라우저 기본 팝업에서 카메라 선택 (TV 등 외부 카메라 지원)

**손가락 인식**:
- 펴진 손가락만 인식 (tip이 pip보다 60% 이상 멀어야)
- 1~2개 펼치면 엄지 제외, 3개 이상이면 엄지 포함
- 손가락 끝에 민트색 글로우 포인터 (카메라 뷰에만 표시)
- 손 감지 신뢰도: 0.75 (오인식 방지)

**포인터 스무딩**:
- fingerTips: lerp 0.15 (EMA) + 2px 미만 지터 무시
- handX/handY: lerp 0.15
- 손 감지 실패 시 5프레임 유예기간 (깜빡임 방지)

**제스처별 반응**:
| 제스처 | 반응 |
|--------|------|
| 손 오므림 | 완전 무반응 (점들 자연 상태) |
| 손 펼침 | 각 손가락 끝에서 반발력 → 파티클 흩어짐 |
| 오므림→쫙 펼침 | 폭발적 산란 (15프레임 burst) |
| 손바닥 좌/우 스와이프 | 이동 방향으로 파티클 밀어내기 |
| 검지 원형 돌리기 | 각 서브존에서 도넛 회오리 |
| 도넛 중 손가락 쫙 | 카오스 폭발 (2초간, 점점 사라짐) |

**카메라 뷰 가이드**:
- Power/Casual/Light × T1/T2/T3 그리드 오버레이
- 손가락 위치한 셀의 T라벨 흰색 100% 활성화
- DPR 스케일링 적용, 리사이즈 시 가이드도 비례

### 13. AR 이펙트 (MediaPipe Face Mesh)
카메라 활성화 시 자동 얼굴 인식 (**최대 3명 동시**):
- **비니**: 머리 위 덮개, 불투명 다크, '현대카드' 흰색 텍스트
- **선글라스**: 아비에이터 스타일, 눈 위치/기울기 자동 추적
- **눈썹**: 굵은 타원형, 선글라스 아래 레이어
- **마스크**: 흰색 90% 크기, 코~턱 영역, 접힌 선 + 귀걸이
- 레이어 순서: AR → 가이드 선 → 핑거팁 (가이드가 AR 위에)

### 14. 상세페이지 진입
**진입 방법 1 — 헤더 MAU 클릭**:
- 헤더 MAU 영역(x:28~28+kW, y:54~96) 위 투명 div 핫스팟
- 호버 시 `cursor:pointer`, 클릭 시 `showDetailPage()` 호출

**진입 방법 2 — 손가락 흔들기 (T1 탭 제스처)**:
- 검지로 T1 영역을 가리킨 상태에서 손가락을 흔들기 (진동)
- tip+MCP 중간점으로 영역 판정 (상하 인식률 균일)
- 영역 진입 후 10프레임 대기 → 흔들림 감지 시작
- Y축 variance(진동량) 기반 감지: 자연 떨림 < 0.00001 < 의도적 흔들기
- 영역 벗어나도 유예기간 (프레임 감소 방식, 즉시 리셋 안 함)

**상세페이지 콘텐츠 — 주간 UV 추이**:
- 검은 배경 풀스크린 오버레이 (z-index 99), 0.4초 페이드인/아웃
- 좌상단 `<` 백버튼 / 콘텐츠: 좌우 74px 마진 (백버튼 텍스트 그리드 기준)
- **KPI 3열**: 당월진행 763만 / 전월동기 749만 / 전월동기대비 ▲14만(green)
- **라인차트**: 이번달(#50d278 실선) + 전월동기(white 점선), 5주차, Y축 550~850만
- **Y축**: 100만 단위 밝게 / 50만 단위 흐리게, 800만(목표)은 green 점선 + "목표→" 레이블
- **크로스헤어 인터랙션**: 마우스 X 이동 시 1일 단위 스냅, 말풍선(날짜+이번달/전월동기 값)
  - 주차 시작일(1, 8, 15, 22, 29일)이 SVG x(80, 375, 670, 965, 1260)에 정확히 매핑
  - 일자 간 보간으로 Y값 계산, 틱 피드백: 모바일 진동 4ms / PC Web Audio 클릭음
- 하단: 범례 + 각주

**복귀**:
- 손바닥 펴고(3개+ 손가락) 왼쪽으로 빠르게 스와이프 (handVX < -8)
- 마우스: 좌상단 백버튼 클릭

### 15. 회오리 + 카오스 시스템
**회오리 (Swirl)**:
- 검지 하나로 원형 돌리기 → 각 서브존 중심 기준 도넛 회오리
- 연속 이동벡터의 외적(cross product)으로 회전 감지
- swirlForce: -1~1 범위, 감쇠 0.97 (멈춰도 서서히 줄어듦)

**카오스 (Chaos)**:
- 도넛 회오리 중 손가락 5개 쫙 펼치면 발동
- swirlEnergy > 0.2일 때 트리거
- 120프레임(2초) 동안 랜덤 폭발력 + 속도 반전 + 바깥 방출
- 점점 감소하며 사라짐

## 주요 함수 맵
| 함수 | 역할 |
|------|------|
| `computeLayout()` | 반응형 레이아웃 계산 |

| `Particle.startMigration()` | 웨이포인트 기반 이동 시작 (채널 경유) |

| `Particle.returnFromChurn()` | 이탈에서 Light로 복귀 |
| `buildGrid() + applyRepulsion()` | 공간 그리드 기반 파티클 상호작용 |
| `tryMigrations()` | 프레임당 마이그레이션 확률 판정 |
| `applyMouseForce()` | 마우스/손 반발력 + burst + swirl + chaos |
| `drawTanks()` | 탱크 테두리 + 구멍 + erase 처리 |
| `drawAllPipes()` | 세로/가로 파이프 + 쉐브론 애니메이션 |
| `drawActionPanel()` | 캐러셀 슬라이드 + 자동/클릭 전환 |

| `showDetailPage() / hideDetailPage()` | 상세페이지 전환 |
