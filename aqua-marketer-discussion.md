## 논의할 사항

### 1. "이상" 뱃지 임계치
- 현재 `diff <= -0.3일`로 하드코딩 (aqua_v0.84.html:2117 / 이후 버전 동일)
- Claude가 임의로 정한 값, 사용자 승인 없음
- **질문**: 마케터 관점에서 "의미 있는 이상 감소"의 기준은? (−0.3이 적절? 더 엄격/완화?)

### 2. 저빈도/활성 경계 수직 점선
- 현재 **제거 상태** (세 패널 모두 같은 위치라 정보량 없다고 판단)
- **질문**: 0–2일 / 3일+ 경계 구분 시각화가 필요한지? 필요하다면 어떤 형태로?

### 3. LF 차트 백그라운드 신호 (존/마커)
- 여러 번 시도: 존 해치, 점 마커, 파도 그라데이션, 대각선 하이라이트 — 전부 "노이즈만 늘린다"고 반려
- 현재는 호버 시 전월↔당월 간극 해치만 남음
- **질문**: 실제 마케터가 LF 차트에서 "어디를 먼저 봐야 할지" 직관적으로 잡기 위해 어떤 시각 신호가 유용한가?

### 4. LF narrative "저빈도/활성" 명명 (v0.85에서 결정)
- aqua_v0.85.html LF 모드 narrative line1에서 **저빈도(1–2일) / 활성(3일+)** 표기 사용
- 사용자(디자이너) 선택: 마케터 스캔 효율 위해 *의미 기반 2그룹*이 *순수 일수 표기* 또는 *3그룹*보다 낫다고 판단
- "활성" 충돌 우려 검토 → 실제 UI에 "활성 사용자"라는 한글 라벨은 노출되지 않음 (Retention Track 영문 라벨만). cueLine "활성 복귀"와 같은 의미 영역 가리켜 narrative 내 일관성 ↑
- 후보 비교:
  - **저빈도/활성** (선택): 친숙·직관, 비대칭이지만 일상 마케터 어휘
  - 저빈도/고빈도 (탈락): 평행 구조 좋으나 "고빈도"가 분석 보고서 톤이라 인지부하 ↑
- CSV/세그 정의에는 등장하지 않는 **narrative 한정 nickname** (괄호 일수 anchor 항상 병기)
- **질문**: 마케터가 실제로 보고 "저빈도/활성"을 직관적으로 이해하나? 다른 표현이 더 익숙한가? (예: 가끔/자주, 덜/많이 등)
- **질문**: 일수 anchor 없이 "활성"만 단독 사용해도 Retention Track 의미와 안 헷갈리는가? (현재는 항상 병기)

### 5. 휴일차이 비교 범위 (v0.86에서 발견, 2026-04-26)
- 현재 로직: 추출일 기반으로 **당월 1~N일 vs 전월 1~N일** 같은 일자 구간 비교 (aqua_v0.86.html:1675~1709)
- 예: 추출일 4/6 → "4. 1 ~ 4. 5" vs "3. 1 ~ 3. 5" → 양쪽 다 평일 3 / 휴일 2 → 차이 0
- 사용자 의문: 진짜 캘린더 기준의 **전월 전체(31일) vs 당월 진행분** 비교가 맞을지?
- **질문**: 마케터가 "휴일 차이"를 볼 때 의미하는 건 (a) 진행 동기간 비교 (현재 방식) 인지, (b) 전월 캘린더 전체와 당월 진행분 비교 인지?
