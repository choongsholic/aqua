### 6. LF narrative 코호트 모델 재정의 (v0.87, 2026-04-27)
**배경**: 사용자(디자이너)가 v0.86에서 Power 카드의 "<<>>전체 세그 규모 확장" narrative를 보고 의문 제기. 세그현황판 Power는 180→180(변화 0)인데 LF 차트 narrative는 "확장"을 단언 → 같은 카드 안에서 정면 충돌.

**핵심 개념 정리 (사용자 설명에 따라)**:
- **세그현황판** = 동적 멤버십. 매주 자격 재평가 ("이번 주 기준 8일+인 사람이 Power"). 전월마감 180만은 비교 기준선. 핵심 시그널 = 당월 Power 규모의 주차별 변화.
- **이탈 조기 경보(LF)** = 고정 코호트. **전월 Power였던 180만을 통째로 가둠**. 그 180만의 당월 로그인일수 분포를 보고 전월 동기 분포와 비교. 모집단 크기는 정의상 180만 고정.

**그래서 v0.86 narrative 로직이 근본부터 잘못됨**:
- "전체 세그 규모 확장/축소"는 LF 코호트 맥락에 존재할 수 없는 개념 (코호트 닫혀있어 합 항상 180)
- "저빈도 X만 증가, 활성 Y만 증가" 같은 인원 증감 단언도 코호트 모델에선 문법 오류 (한쪽 +면 다른 쪽 −여야 정상)

**v0.87 적용 (aqua_v0.87.html, line 2319~2391)**:
- 출처는 그대로 seg_visit (LF 전용 데이터) 유지
- "확장/축소" 분기 제거
- 버킷 간 이동 표기로 변경 — "X만명이 [origin] → [destination] 이동"
- 6가지 흐름 패턴 분기: 활성→휴면(강한 이탈), 활성→저빈도(빈도 하락), 저빈도→휴면(휴면화), 휴면→활성(복귀), 휴면→저빈도(가벼운 복귀), 저빈도→활성(빈도 상승)
- 코호트 합 어긋남(±5만 초과)이면 시그널 보류 + ⚠ "데이터 검증 필요" 경고
- `.lf-data-warn` CSS 추가 (clr-warn 노란색)

**질문 (마케터 확인용)**:
- 6가지 이동 패턴 명명이 마케터 직관과 맞나? (특히 "강한 이탈 시그널" / "저빈도 휴면화" 표현)
- "X만명이 활성(3일+) → 휴면(0일) 이동" 같은 이동 표기가 인원 증감 표기보다 이해되는가?
- NOISE 임계 5만이 적절한가? (5만 미만 변동은 노이즈로 무시)
