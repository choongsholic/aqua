# DP-AQUA Growth 프로젝트 컨텍스트

## 작업 파일
- `dp-aqua_growth.html` — 메인 대시보드 파일 (dp-aqua_mau.html에서 rename)

## 프로젝트 목적
현대카드 앱 MAU 800만 달성을 위한 유저 행동 분석 시각화 대시보드.
단일 HTML 파일, Canvas 2D 기반 실시간 파티클 시뮬레이션.

## 대시보드 구조

### 전체 레이아웃
- **헤더** (96px): DP-AQUA 타이틀 + KPI (MAU 763만 / Power 180만 / Casual 300만 / Light 280만 / 이상징후)
- **좌측**: 신규유입존 (제거 검토 중)
- **중앙**: USER AQUARIUM (메인 파티클 시뮬레이션)
- **우측**: 신호 패널 / 액션 패널 / 이탈존 (제거 검토 중)

## 세그먼트 정의

### MAU 트랙 (흰색 아웃라인 버블 파티클)
| 세그먼트 | 규모 | 정의 |
|---------|------|------|
| 미설치자 | 113만 | 신규 카드 발급 후 앱 미설치/미로그인 |
| 장기 미이용 | 110만 | 3개월 초과 미방문 |
| 간헐 이용 | 190만 | 3개월 내 비연속 이용 |

### Retention 트랙 (건강 상태별 색상 채워진 파티클)
| 세그먼트 | 규모 | 정의 |
|---------|------|------|
| Light | 280만 | 월 1~2회 이용 |
| Casual | 300만 | 월 3~7회 이용 |
| Power | 180만 | 월 8회 이상 이용 |

## 이탈(Churn) 정의 — 확정

**이탈 = 앱 MAU 이탈 = 해당 월 앱 방문 0회**

- Light 유저(월 1~2회)가 그 달 한 번도 앱을 안 연 상태
- 실무 기준: 30일 미방문
- Power/Casual에서 직접 이탈은 현실에서 거의 없음 → Light 경유가 정상 경로
- 이탈 단계: Light → (MAU 이탈) → 재활성화 or 완전 이탈(카드 해지)

---

## 이동 구조 (현재 구현)

### MAU 트랙 내부
- 장기미이용 ↔ 간헐이용: 수평 파이프(pipes.di)로 연결, 양방향 간헐 이동

### MAU → Retention 전환
- 미설치자 / 장기미이용 / 간헐이용 → Light
- 경로: MAU 탱크 **하단 구멍** → 분리대 통과 → Light **상단 구멍** → 탱크 내부
- 확률: 미설치자 0.3%, 장기미이용 0.8%, 간헐이용 1.4% (프레임당)

### Retention 트랙 내부
- Light ↔ Casual ↔ Power: 수평 파이프(pipes.lc, pipes.cp)로 연결
- 업그레이드: healthy/good 상태만 / 다운그레이드: caution/danger 상태만

### 이탈 경로
- **Light → 이탈**: Light 하단 구멍 → 아쿠아리움 하단 채널(pipes.light_churn) → 이탈존
- Casual → 이탈: 극소량 (원래 대비 ×0.06)
- Power → 이탈: 거의 없음 (원래 대비 ×0.02)

### 신규 유입 경로
- **65%** → Light (new_in 파이프 → Light 좌측 구멍)
- **35%** → 미설치자 (newZone → 미설치자 하단 구멍 진입)

## 신규유입존 / 이탈존 제거 검토 중

### 현재 구현 상태
- **신규유입존**: 아쿠아리움 좌측 별도 박스, 파티클이 잠깐 머물다 이동하는 경유지
- **이탈존**: 아쿠아리움 우측 별도 박스, fade-out 소멸 지점

### 제거 근거
- 신규유입/이탈은 "세그먼트"가 아닌 흐름(flow) — 분석 대상이 아님
- 대시보드 주인공은 6개 탱크, 사이드 박스가 오히려 노이즈
- 제거 시 대안:
  - 신규 파티클 → Light 좌측 벽 or 미설치자 하단에서 페이드인으로 등장
  - 이탈 파티클 → Light 하단 구멍에서 빠져나가며 페이드아웃
- 아쿠아리움 좌우 공간 확보 → 탱크 더 넓게 사용 가능

### 결정 보류
→ 제거 방향으로 기울어져 있으나 최종 미확정

## 기술 구현 핵심

### 파티클 시스템
- Particle 클래스: segment, health, radius, waypoints, migrating 상태 관리
- MAU 트랙: 흰색 아웃라인(stroke) 버블 / Retention 트랙: 건강 상태별 색상(fill)
- 건강 상태: healthy(녹색) / good(파란) / normal(회색) / caution(노랑) / danger(빨강)
- 총 파티클: ~5,860개 (1:2000 스케일)

### 레이아웃 전역 변수
```javascript
let mauSectionH = 0, trackSepGlobal = 16; // computeLayout에서 설정
aquarium._mauHoleCX = { uninstalled, dormant, intermittent } // MAU 하단 구멍 X좌표
aquarium._lightHoleCX        // Light 상단 구멍 (MAU→Light 진입)
aquarium._lightChurnHoleCX   // Light 하단 구멍 (이탈 출구)
```

### 파이프 객체
```javascript
pipes.lc          // Light ↔ Casual 수평 파이프
pipes.cp          // Casual ↔ Power 수평 파이프
pipes.di          // 장기미이용 ↔ 간헐이용 수평 파이프
pipes.new_in      // 신규유입 → Light
pipes.churn_out   // Power → 이탈존 (Casual/Power 극소량용)
pipes.light_churn // Light 하단 → 이탈존 수평 채널
```
