# 보고 문서 작업 템플릿

dashboard.html을 디자인 베이스로 새 보고 문서를 만들 때의 작업 방식. 사용자가 요건서만 던지면 Claude가 이 흐름대로 처리.

---

## 입력 — 사용자가 주는 것

**요건서.md 형식**으로 콘텐츠만 정리해서 전달. 형식 예시는 `요건서.md` 참고.

```
## 1장

[문서 제목]
...
[팀명]
...
[날짜]
...

## 2장

[장표 제목]
...
[설명 (불릿)]
- ...
- ...

## 3장

[장표 제목]
...
[3카드]
- 카드1: ...
- 카드2: ...
- 카드3: ...

(이하 N장까지 자유롭게)
```

### 사용 가능한 라벨 (예시)
- `[문서 제목]` `[팀명]` `[날짜]` (커버 전용)
- `[장표 제목]`
- `[부제]`
- `[중앙 대문장]`
- `[설명 (불릿)]` — 일반 항목, `(강조)` 표시 가능
- `[N카드]` — 카드 N개 가로 배치
- `[검정 헤더 N카드]` — head + body 구조
- `[2열 박스]` / `[1열 박스]` — bhx-card
- `[2열 박스 (RDI 형태)]` — 가운데 lede + 조건 + 공식
- `[BA 3카드]` — head + lede + 항목 리스트
- `[4패턴 카드]` — OOO/XOO 같은 코드 + 상단/하단 텍스트
- `[비교 박스]` — 골드 톤 비교 행
- `[단계 표시 (N stages)]`
- `[공식]` `[공식 노트]`
- `[이미지]` — 파일명 + 설명, 받기 전엔 placeholder
- `[이미지 좌우 2분할]` `[좌우 분할 — 좌:이미지+캡션, 우:리스트]`
- `[표 — N컬럼 × N행]`
- `[하단 메시지]` `[하단 푸터]`
- `[큰 텍스트 중앙]`
- `[N 라벨]` — 인라인 뱃지 (예: `[논의 필요 라벨]`)

**새로운 형태 필요하면 자유롭게 추가.** Claude가 보고 적절한 컴포넌트 매핑 또는 신규 컴포넌트 추가.

### 추가로 알려주면 좋은 정보 (선택)
- 청중 (상사 / 팀 / 외부)
- 톤 (진행보고 / 결론형 / 논의용 / 의사결정 요청)
- 강조하고 싶은 슬라이드 / 메시지

---

## 처리 — Claude가 하는 일

1. **파일 복제** — `dashboard.html`을 새 파일명으로 복제 (예: `q1-report.html`, `marketing-plan.html`)
2. **localStorage 키 rename** — `dashboard-edits-v3` → `[새파일명]-edits-v1` 등으로 분리 (편집 캐시 충돌 방지)
3. **카운터/네비게이션 자동 갱신** — `5 / 27` 같은 카운터를 새 슬라이드 수에 맞춰 변경
4. **콘텐츠 매핑**:
   - 같은 패턴이면 `guide.md`의 기존 컴포넌트 그대로 재활용
   - 새 패턴이면 같은 디자인 토큰(`--accent`, `--text` 등)으로 신규 컴포넌트 추가
   - 애니메이션 `delay-1~8` 자동 분배
5. **이미지** — 자료 받기 전엔 placeholder. 받으면 `imgs/` 폴더에 임포트
6. **폰트** — `../fonts/` 경로로 공용 폰트 폴더 참조 (별도 설정 불필요)

### Claude 자유 재량
- 카드 수 / 레이아웃 (2열 / 3열 / 4열) 적절히 분배
- 애니메이션 delay 분배
- 색상 강조 위치 (`--accent` 사용 빈도)
- bullet 강조 (`<li class="hl">`)
- 슬라이드 컴포넌트 자동 선택

### Claude 사용자 확인 필수
- 슬라이드 개수 추가/삭제
- 의미 요약 / 재작성 / 표현 변경
- 대규모 재구조 (섹션 합치기 / 분리)
- 새 컴포넌트 도입 (기존으로 안 풀리는 경우만)

---

## 출력 — 결과물

- 같은 폴더(`Document_templete/[새이름]/`)에 새 HTML
- 디자인 톤 dashboard.html과 일관 (다크/라이트 토글, 키보드 네비, 인라인 편집, PNG 다운로드 등 시스템 기능 모두 동작)
- 폰트는 `../fonts/` 공유

---

## 참조 파일

- `dashboard.html` — 디자인 시스템이 살아있는 실제 슬라이드 (베이스)
- `guide.md` — 디자인 토큰 / 컴포넌트 / 사용 패턴 레퍼런스
- `요건서.md` — 27장 콘텐츠 요건서 샘플 (형식 참고)
- `템플릿.md` — 이 문서 (작업 방식 정리)
